# X Posting Growth

## Overview
用最少字数讲清价值、证据与下一步，目标是吸引关注与互动，而不是堆功能。

## Inputs
- 进展要点（3-5 条，必须可验证）
- 目标受众（开发者/产品/用户）
- 目标（涨粉/招募/求反馈）
- 链接（可选，放文末）
- 日期（必填）

## Workflow
1) 选一个主角度：**结果** / **变化** / **开源** / **可用性** / **进度节奏**。只选一个。
2) 写一句钩子：用“从0到1”“今天完成”“我们解决了X”开头，避免空泛。
3) 给证据：用 2-4 条短句列出具体改动或产出。
4) 写行动：明确让人做什么（关注/回复/试用/提意见）。
5) 精简：去掉所有可删词，尽量一屏内读完。

## Patterns
**单条进展贴（推荐）**
```
YYYY-MM-DD
一句话钩子（结果/变化）
• 具体进展 1
• 具体进展 2
• 具体进展 3
CTA（关注/回复/试用）
链接（可选，文末）
```

**里程碑公告贴**
```
YYYY-MM-DD
我们完成了 X（里程碑）
影响/好处（1 句话）
证据 1-2 条
CTA + 链接
```

## Micro Rules
- 每条只讲一个主线，不要堆 10 个功能。
- 用具体名词：页面/命令/接口/版本号，而不是“优化/完善/升级”。
- 链接放最后；能不放就不放。
- 少量表情即可（0-2 个），不刷屏。
- 结尾必须有 CTA。

## Common Mistakes
- 只讲愿景没有证据 -> 必须列出可验证产出。
- 过长/多主题 -> 删到 4-6 行。
- 没有 CTA -> 会显著降低互动。

## Example (ClawBay)
```
2026-02-05
从0到可用：ClawBay 现在能真正让 Claw 上线并对话。
• 注册页支持插件版/无插件版提示词切换
• 新增 skill-no-plugin.md，无需插件也能接入
• 认领码闭环 + streaming chat + 错误提示更清晰
关注我，接下来每天更新进展。clawbay.ai
```
